## ==图==

- 对于n个顶点的图G=(V,E)，可用n×n的矩阵来表示，矩阵中每个元素定义为：

  *A*[*i*, *j*] =1，若(vi,vj)或<vi,vj>是E中的边或弧；0，反之。

  ###  一、图的基本概念

  #### 定义

  图（Graph）由**顶点集合 V** 和 **边/弧集合 E（或 A）** 构成：

  G=(V,E) 或 G=(V,A)G = (V, E) \text{ 或 } G = (V, A)G=(V,E) 或 G=(V,A)

  - **无向图**：边是无序对 `(vi, vj)`
  - **有向图**：弧是有序对 `<vi, vj>`，有方向性
  - **网（带权图）**：边或弧上带有权值（表示距离、时间、花费等）

  ------

  #### 常用术语

  | 名称             | 含义                                         |
  | ---------------- | -------------------------------------------- |
  | **子图**         | G' 的顶点和边均属于 G                        |
  | **度（Degree）** | 无向图中与顶点相连的边数                     |
  | **入度 / 出度**  | 有向图中以该顶点为终点/起点的弧数            |
  | **路径**         | 顶点序列 (v₁, v₂, …, vk)，边依次相连         |
  | **路径长度**     | 路径上边（弧）的数目                         |
  | **连通图**       | 无向图中任意两点连通                         |
  | **强连通图**     | 有向图中任意两点互通（方向可达）             |
  | **完全图**       | 任意两顶点间都存在边；无向图有 n(n−1)/2 条边 |
  | **完全有向图**   | 每对顶点间存在两个方向的弧，共 n(n−1) 条     |

  ------

  ### 二、图的存储结构

  #### 邻接矩阵（Adjacency Matrix）

  - 用 `n × n` 矩阵 A 表示：

    A[i][j]={1,(vi,vj)∈E0,其他情况A[i][j] =  \begin{cases} 1, & (vi, vj)\in E \\ 0, & 其他情况 \end{cases}A[i][j]={1,0,(vi,vj)∈E其他情况

  - 若为网（带权图）：`A[i][j] = wij`，否则为 0。

  - **特点：**

    - 查找边快（O(1)）
    - 空间复杂度 O(n²)
    - 适合稠密图
    - 无向图矩阵对称，可压缩存储

  ------

  #### 邻接表（Adjacency List）

  - 每个顶点建立一个单链表存储与其邻接的顶点。
  - 每个链结点含有：
    - 邻接顶点序号
    - 边权值（可选）
    - 指针域
  - **特点：**
    - 空间效率高（O(n + e)）
    - 适合稀疏图
    - 查找相邻顶点方便

  ------

  ### 三、图的遍历算法

  #### 深度优先搜索（DFS）

  **思想：**
   从起点出发，尽可能深入访问未访问的邻接点，访问到底后回溯。

  **递归形式：**

  ```
  void DFS(int A[n][n], int n, int v, int visited[]) {
      printf("%d ", v);
      visited[v] = 1;
      for (int i = 0; i < n; i++)
          if (A[v][i] == 1 && !visited[i])
              DFS(A, n, i, visited);
  }
  ```

  **性质：**

  - 类似树的先序遍历
  - 常用于求连通分量、拓扑排序、判断连通性

  ------

  #### 广度优先搜索（BFS）

  **思想：**
   从起点开始，先访问所有直接邻接点，再访问这些邻接点的邻接点（层次扩展）。

  **实现：**
   用队列记录已访问的顶点。

  假设我们有一个图，顶点0、1、2，其中顶点0的邻接点是顶点1和2。
  那么adj[0].vexdata = 0，adj[0].firstarc指向一个节点链：节点1（data=1）和节点2（data=2）。
  
  ```
  void BFS(HEADNODE adj[], int n, int v, int visited[]) {
      int queue[MAX], front = -1, rear = 0;
      NODE *p;
      printf("%d ", adj[v].vexdata);
      visited[v] = 1;
      queue[rear] = v;
      while (front != rear) {
          front++;
          v = queue[front];
          p = adj[v].firstarc;
          while (p) {
              if (!visited[p->data]) {
                  printf("%d ", adj[p->data].vexdata);
                  visited[p->data] = 1;
                  queue[++rear] = p->data;
              }
              p = p->next;
          }
      }
  }
  ```
  
  ------
  
  ### 四、拓扑排序

  **定义：**
   对一个有向无环图（DAG），若存在一个顶点序列，使得对任意弧 `<vi, vj>`，vi 都排在 vj 之前，则称为**拓扑序列**。
  
  **算法步骤：**
  
  1. 找出入度为 0 的顶点，输出并入栈；
  2. 删除该顶点及所有出边；
  3. 重复直到所有顶点输出完毕。
  
  **应用：**
  
  - 任务依赖调度
  - 课程先修关系分析
  - 编译依赖排序





## 查找

- 用比较次数的平均值，即统计意义上的数学期望值来评估查找算法，称为平均查找长度ASL
- **线性查找**/**顺序查找**
  - ASL=(n+1)/2

- **对分查找**

  - 是对有序表
  - ASL=Log2(n+1)-1

- **分块查找**

  - 对索引表对分或顺序查找,以确定所在块。后一块中的所有关键字均大于前一块中最大的关键字
  - 在所在块中进行顺序查找

- **二叉排序树查找**

  - 既查找又插入的过程称为动态查找

  - 若找到，返回 `1`；若没找到，则在合适位置插入结点 `k`，并返回 `0`。

  - 要求：对二叉排序树进行**中序遍历**，得到的序列一定是**递增有序**的。

  - 最坏：asl=(n+1)/2  最好：log2(n)

  - ```
    int BstSearch(NODE *t,int k)
    { 	NODE *p,*q,*s;
    	p=t;
    	while (p){
    		if (k==p->data) return(1);
    		if (k<p->data){ q=p; p=p->lchild; }
    		if (k>p->data){ q=p; p=p->rchild; }
    	}
    	s=GetNode(); if (!s) return(0);
    	s->data=k; s->lchild=s->rchild=NULL;
    	if (t==NULL) p=s;
    	else if (k<q->data) q->lchild=s;
    	else q->rchild=s;
    	return(0);
    }
    ```

## 排序

- **分类**：排序时待排序记录存放在内存中进行排序，则称为内部排序；若待排序记录数量很大，在内存中一次不能容纳全部记录，需要在排序过程中对外存进行访问，则称为外部排序

- **选择排序**

  ```
  SelSort(int r[],int n)
  {for (i=0;i<n-1;i++){
  	j=i;//记下位置
  	for (k=i+1;k<n;k++) if(r[j]>r[k]) j=k;//和余下的比较
  	if (j>i){ t=r[j]; r[j]=r[i]; r[i]=t; } //交换
  }}
  ```

  - 比较n(n-1)/2次
  - 总的时间复杂度为：O(n^2) 空间复杂度O（1）

- **堆排序**

  - 可以排序的完全二叉树
  - 将现有的序列构成一个堆，输出堆顶元素，重新调整元素，构成新的堆，直到堆空为止
  - 怎么输出堆顶元素：设完全二叉树r[1..n]中结点r[k]的左右子树均为堆，为构成以r[k]为根结点的堆，需进行调整。方法是将r[k]的值与其左右子树的根结点进行比较，若不满足堆的条件，则将r[k]与其左右子树中根结点大的交换，继续进行比较，直到所有子树均满足堆为止。

  ```
  Sift(int r[],int n,int k) //调整r[k]结点，让r[k]最值
  { 	int i,j,t; //使其成为堆
  	i=k; j=2*i+1; t=r[i];
  	while (j<n){
  		if (j<n-1&&r[j]<r[j+1]) j++;  左<右
  		if (t<r[j]){ r[i]=r[j]; i=j; j=2*i+1; } //j指向更大的孩子;继续向下调整
  		else break;
  	}
  	r[i]=t;
  }
  ```

  - 对于一个无序序列r[0..n-1]构成的完全二叉树，只要从它最后一个非叶子结点[(n-2)/2]开始直到根结点为止，逐步进行上述调整，即可将该完全二叉树调整为堆。
    - 当 `2*i + 1 < n` 时，结点 `i` 至少有左孩子（是非叶子结点）

  ```
  BSift(int r[],int n)  //构造大堆
  {
  	int p=(n-2)/2,i;
  	for (i=p;i>=0;i--) Sift(r,n,i);
  }
  HeapSort(int r[],int n)
  { 	int i;
  	BSift(r,n); //构造堆
  	for (i=n-1;i>=0;i--){
  		swap(&r[0],&r[i]); //最大的放在最后，升序排列
  		Sift(r,i,0); //调整其余的元素为堆
  	}
  }
  ```

  - 时间复杂度：O(nlog2n)         空间复杂度O（1）
  - 对记录数目较少的不提倡堆排序算法，但当记录较多则比较有效
  - 算法不稳定

- **插入排序**

  - t前面是排序好的；j是已排序区最后一个元素；从右往左扫描“已排序区”

  ```
  InsertSort(int r[],int n)
  { 	for (i=1;i<n;i++){
  	t=r[i]; j=i-1;
  	while (t<r[j]){ r[j+1]=r[j]; j--; }//依次向后移动，找到插入位置
  	r[j+1]=t;
  	}
  }
  ```

  - 时间复杂度：O(n^2)         空间复杂度O（1）
  - 算法是稳定的

- **希尔排序**

- **交换排序**：**冒泡排序**

  - ```
    void Bsort(int r[], int n)
    {
    	int j, k, f=n-1; //f初值为最后一个元素
    	while(f>=0) { //表示上遍扫描发生过交换
    		k=f-1; f=-1; //f~n-1已排好序
    		for(j=0;j<=k;j++) {
    			if(r[j]>r[j+1])
    				{swap(&r[j],&r[j+1]);f=j;}

    ```

  - 最好情况下时间复杂度为O(n)，最坏时间复杂度：O(n^2) 

- **交换排序**：**快速排序**

  - 对冒泡排序的一种改进

  - 任取线性表的某个元素作为基准，将整个对象序列划分为左右两个子序列：左侧子序列中所有元素都小于或等于基准，右侧子序列中所有元素都大于基准

  - ```
    QkSort(int r[],int low,int high)

    		while (i<j&&r[j]>=x) j--;

    		r[j]=r[i];
    	}
    	r[i]=x;
    	QkSort(r,low,i-1); 
    	QkSort(r,i+1,hig);
    }
    ```

## 处理器管理

- 多道程序系统中，处理器数小于程序数，于是处理器就在各程序间进行切换，因此在一个处理器上要交叉运行若干道程序。处理器管理就是要解决用户递交的作业何时调入内存，在调入内存的各个作业程序间如何分配处理器，以达到各道程序能协调一致运行，而系统资源又能得到最大程度的利用。

- **作业和进程**

  - 作业调度（宏观）并发（串联）         进程调度（微观）并行（并联）
  - 进程调度仅负责CPU分配，而作业调度要负责CPU之外的资源。
  - 一个C语言程序要经历编辑、编译、连接、运行四个作业步
  - 进程：动态 有生命过程         >          线程（多线程，迅雷
  - 进程的三种状态：运行，就绪，阻塞/等待。进程从执行状态转为就绪状态通常是由于时间片用完（在分时系统中），或者被更高优先级的进程抢占。等待I/O或申请资源失败会导致进程进入阻塞状态，进程终止会进入终止状态。
  - 处于阻塞状态的进程是绝不可能叫醒它自己的。

- **特权指令、处理器状态**

  - **1.** 特权指令：只能由操作系统使用。

    **2.** 非特权指令： 供一般用户使用。

  - 从Intel 80386开始，出于安全性和稳定性的考虑，该系列的CPU可以运行ring0（0级环）至ring3(3级环)从高到低四个不同的权限级，对数据也提供相应的四个保护级别。运行于较低级别的代码不能随意调用高级别的代码和访问较高级别的数据，只有ring0 层的代码才可以对物理硬件直接进行访问。

  - 脱机输入/输出(Off-Line I/O)

  - **单、多道批处理系统**
  
    - 单道批处理系统中，内存中仅有一道作业
  
- 一个作业从进入系统到运行结束，一般要经历以下四种状态: **提交** **→** **收容** **→** **执行** **→** **完成**

- 进程数目总是大于CPU数目。

- 进程控制块 PCB )记录进程的有关信息，标志进程的存在

  - Linux中：pid进程ID

- 进程控制：原语实现。原语在执行期间是不允许被中断的。

  - 创建进程原语：申请空白PCB
  - 终止
  - 阻塞
  - 唤醒

- 线程

  - 在进程的创建、撤销、切换中，系统必须为之付出较大的时空开销，导致了系统中进程的数量不宜过多，进程切换的频率不宜过高，但这也就限制了并发程度的进一步提高。
  - 对作为调度和分派的基本单位，不同时作为独立分配资源的单位；对拥有资源的单位，不对之进行频繁切换。
  - 线程是进程中的一个实体，是被系统独立调度和分派的基本单位。
  - 线程自己基本不拥有系统资源，只拥有少量必不可少的资源：程序计数器、一组寄存器、栈。

  - 先来先服务调度算法：🚻排队
  - 优先级调度算法
  - 时间片轮转算法：每次尿一点

- **多道程序并发运行出现的问题**  重要哦

  - “同步” 司机售票员配合

  - “互斥”共享资源，竞争关系

  - 解决方案：

    - 同步原语对某信号量进行操作以实现同步与互斥的方法，通常称为==P-V操作==。
    - s：可用资源数
    - V：s=s+1,s<0阻塞。操作意味着进程释放一个单位的可用资源，释放资源
    - P：s=s-1,s<=0阻塞。s=0就不能拿了。成对出现，申请资源
    - 有P就有V，就像有借有还
    - 等待事件前 → `P(S)`

      事件完成后 → `V(S)`

      临界区前后 → `P(mutex)` / `V(mutex)`
    - 用 P 等待条件，用 V 释放条件
    - 在以下场景中信号量的初值应如何设置，并简要说明理由：

      1. 用于互斥的信号量  1
      2. 用于同步的信号量（进程A要在进程B之后执行）  0
      3. 表示资源数量的信号量（有5个同类资源）  5

  - 双向制约

    - 生产者和消费者。防止生产者将物品放入已满的缓冲区；禁止消费者从空缓冲区取物品。

      ```
      生产者
      L1: {生产物品}        // 先做个包子（跟蒸笼无关）
      P(s1)                // 看看蒸笼有没有空位
      {将物品放入缓冲区}     // 有空位就放进去
      V(s2)                // 大喊一声："有包子啦！"
      Goto L1

      读者() {

          readcount++;        // 增加读者计数
          if(readcount == 1)  // 如果是第一个读者

          V(mutex);          // 释放readcount锁
          
          // 读数据（多个读者可同时执行这里）
          
          P(mutex);          // 锁住readcount  
          readcount--;       // 减少读者计数
          if(readcount == 0) // 如果是最后一个读者
              V(w);          // 允许写者进入
          V(mutex);          // 释放readcount锁
      }
      
      // 写者进程
      写者() {
          P(w);              // 申请独占访问权
          // 写数据（只有一个写者可执行这里）
          V(w);              // 释放访问权
      }
